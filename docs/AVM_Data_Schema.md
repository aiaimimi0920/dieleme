# 法拍房时空估价引擎 (AVM) 数据结构与采集规范

本文档定义了构建“法拍房精准自动估价系统(AVM)”所需二次采集的标准化数据格式，以及供 AI 进行零样本/少样本信息抽取的提示词 (Prompt) 模板。

> 本文档为**目标规划契约**。当前线上采集字段与本规范存在差异属于预期现象，当前阶段仅进行规划对齐，不要求立即改动现网代码。

## 1. 核心数学模型字段 (强结构化数据)

这些字段直接参与数值运算（计算距离、计算加权均价、拟合时间折旧曲线），**必须精确无误**。

```json
{
  "item_id": "Long/String",      // 规范层唯一ID（Canonical ID，默认与平台ID一致）(P0)
  "source_item_id": "Long/String",// 原始来源平台ID（迁移期保留，用于回溯）(P0)
  "latitude": "Float",           // 标的物纬度 (P0) - 核心，用于3km筛选
  "longitude": "Float",          // 标的物经度 (P0) - 核心
  "transaction_price": "Float",  // 纸面确切成交价（落槌价），单位：元 (P0) - 基础价格
  "actual_paid_price": "Float",  // 实际支付总价预估（落槌价 + 预估税费 + 拖欠物业水电费），单位：元 (P0) - 真实成本
  "area_sqm": "Float",           // 建筑面积，单位：平方米 (P0)
  "auction_date": "DateTime",    // 拍卖结束时间，格式 YYYY-MM-DD HH:mm:ss (P0)
  "auction_round": "Integer",    // 拍卖轮次：1(一拍) / 2(二拍) / 3(变卖) (P1)
  "housing_type": "String",      // 房屋用途：枚举[住宅, 商业, 办公, 工业, 别墅, 车位, 其他] (P0)
  "bid_count": "Integer",        // 总出价次数 (P1) -> 代表该次价格的挤水分程度
  "apply_count": "Integer"       // 报名总人数 (P1) -> 代表市场热度
}
```

### 1.1 规范层字段约束（必须遵守）

- 主键约束：`item_id` 为规范层主键，`source_item_id` 为来源层回溯字段。
- 单位约束：所有价格字段统一为**元**；仅展示层允许临时换算为万元。
- 坐标约束：`latitude`=纬度，`longitude`=经度，不允许反置。
- 时间约束：统一使用北京时间字符串 `YYYY-MM-DD HH:mm:ss`。

## 2. 需大模型 (AI) 提取的特征字段 (非结构化转换)

这些字段隐藏在长篇的《拍卖公告》、《竞买须知》及评估报告中，需要通过大语言模型进行信息抽取 (Information Extraction, IE)。它们用于对“标准均价”进行折溢价修正，或直接作为异常雷区剔除（如有人强占）。

```json
{
  "community_name": "String",      // 小区标准名称 (P0) -> 同小区赋予极高估价权重
  "build_year": "Integer",         // 建成年份 (P1) -> 计算房龄折旧
  "total_floors": "Integer",       // 总楼层数 (P1)
  "floor_level": "String",         // 所在楼层：枚举[低区, 中区, 高区, 顶层, 底层, 独栋] (P2)
  "has_elevator": "Boolean",       // 是否有电梯 (P1) -> 老公房重要溢价因子
  "orientation": "String",         // 朝向：枚举[南, 南北, 东, 西, 北, 未知] (P2) -> 影响居住体验溢价
  "land_right_type": "String",     // 土地性质：枚举[出让, 划拨, 未知] (P1) -> 划拨地在过户时需补交高额土地出让金，严重影响到手价
  "is_occupied": "Boolean",        // 是否被占用/有人居住并未腾空 (P0) -> 致命剥离因子
  "has_long_lease": "Boolean",     // 是否存在长期租约且买卖不破租赁 (P0) -> 致命剥离因子
  "clear_delivery": "Boolean",     // 法院是否负责清场/交付 (P0) -> false 则为重大折价因子
  "tax_burden": "String",          // 税费承担：枚举[买受人承担全部, 各自承担, 未知] (P1) -> 影响最终到手成本
  "is_haunted": "Boolean",         // 是否涉及刑事案件或为凶宅 (P0) -> 物理隔离因子
  "has_keys": "Boolean",           // 法院是否有钥匙/能否看房 (P2) -> 盲盒房产有较高风险折价
  "property_fee_owed": "Boolean",  // 是否拖欠大额物业/水电费 (P2) -> 隐性成本
  "special_school_tag": "Boolean", // 公告/标题中是否特别强调“学区”、“带学位” (P2) -> 核心溢价项
  "evaluation_price": "Float",     // 法院评估价/市场价 (万元) (P0) -> 计算溢价/折价率的基准
  "layout": "String",              // 户型：如“3室2厅2卫” (P1) -> 同面积下不同户型差价极大
  "is_restricted_purchase": "Boolean", // 是否限购 (P1) -> 不限购的法拍房往往附带极高的“房票溢价”
  "includes_parking": "Boolean",   // 是否带车位一起拍 (P1) -> 剥离车位价值后才能算准房屋纯净单价
  "is_fractional_share": "Boolean",// 是否为按份共有/部分产权 (P0) -> 致命剥离因子，不可交易标的
  "tax_is_company_owned": "Boolean",// 原产权人是否为企业/公司 (P0) -> 极高额的土地增值税预警
  "has_lease_before_mortgage": "Boolean" // 是否为“先抵后租”可被法院强制清场的假租约 (P0) -> 隐藏的极限套利机会
}
```

### 2.1 风控抽取结果的可审计扩展字段（建议）

为降低抽取幻觉风险，建议在每次抽取结果中增加以下扩展字段：

```json
{
  "extraction_confidence": "Float(0~1)",
  "evidence_span": "String/Array",
  "evidence_source": "String(公告|须知|评估报告|页面主文)",
  "extraction_version": "String"
}
```

## 4. 大模型特征抽取标准提示词 (Prompt Template)

在后台的 Python 处理任务中（例如 `submit_task` 调用模型时），请使用以下结构化的提示词来要求模型阅读 HTML 源码并输出纯净的 JSON 结果。

### 系统 Prompt (System Message)
> 你是一个专业的真实房地产估价师与法拍房风控专家。你的任务是从法院复杂的拍卖公告、须知以及页面详情中，像侦探一样精准提取出房屋的核心属性与潜在风险（雷区）。你必须保持绝对的客观，如果文中没有提及某项信息，请将其对应的值设置为 `null`。

### 用户 Prompt (User Message)
> 请仔细阅读以下法拍房网页的文本内容，帮我提取以下结构化字段。
> 
> **提取规则**：
> 1. `community_name`：尽最大努力从地址中提取其规范的小区/楼盘名称（不含具体的门牌号），例如将“上海市浦东新区张江镇郭守敬路111弄xx号”提取为“张江汤臣豪园”。无法判断则提取为 null。
> 2. `build_year`：房屋建成年份，提取纯数字（如 2010）。如果没写，请尝试根据周边楼盘或证号年份推测，无法确定则返回 null。
> 3. `total_floors`：这栋楼一共有多少层。
> 4. `floor_level`：该房产所在的楼层，请归一化为 ["低区", "中区", "高区", "顶层", "底层", "独栋"]。
> 5. `has_elevator`：是否带电梯。true/false。
> 6. `orientation`：房屋的主要朝向。归一化为 ["南", "南北", "东", "西", "北", "未知"]。
> 7. `land_right_type`：土地权利性质。如果写了“出让”返回"出让"，如果写了“划拨”返回"划拨"，否则返回"未知"。注意，划拨极其危险需要补交土地出让金。
> 8. `is_occupied`：房屋目前是否有人居住、占用、或者未腾空状态？如果是，返回 true，否则 false。
> 9. `has_long_lease`：公告中是否提到“带租约”、“设立了租赁权”等？如果是，返回 true。
> 10. `clear_delivery`：法院是否明确表示“负责清场”、“按现状交付且已腾空”？如果是返回 true。如果写着“买受人自行腾退”、“自行解决”、“法院不负责交付”，则务必返回 false！
> 11. `tax_burden`：历史欠费和交易税费是由谁承担？归一化为 ["买受人承担全部", "各自承担", "未知"]。当写明“标的物转让登记手续所涉及的一切税费及明确或不明确的欠费均由买受人承担”时为买受人承担全部。
> 12. `is_haunted`：公告是否提到“发生过非正常死亡”、“涉嫌刑事案”等凶宅特征？如果是，返回 true。
> 13. `housing_type`：这套房子的用途是什么？归一化为 ["住宅", "别墅", "商业", "办公", "工业", "车位", "其他"]。
> 14. `has_keys`：法院是否持有钥匙？是否能正常安排看样？如果是返回 true，如果写明“无钥匙”返回 false。
> 15. `property_fee_owed`：公告中是否提及存在（或可能存在）物业费、水费、电费欠缴？提到了就返回 true。
> 16. `special_school_tag`：公告中是否把“带学位”、“学区房”、“对口XX小学”作为卖点提及？如果是返回 true。
> 17. `evaluation_price`：法院给出的“评估价”或“市场价”是多少万元？请提取纯数字（如果原文是2300000元，请转换为 230）。如无则返回 null。
> 18. `layout`：房屋的户型结构。提取如“3室2厅1厨2卫”这样的格式。找不到则返回 null。
> 19. `is_restricted_purchase`：公告中是否明确标明该房产“受当地限购政策限制”或“需具备购房资格”？如果是真正的限购，返回 true；如果不限购或没提，返回 false。
> 20. `includes_parking`：此次拍卖的标的物，是否**附带了真实的地下车位/车库**一起拍卖？（注意：不是指小区内有公共停车位，而是这个拍品本身包含了车位产权或使用权）。如果是，返回 true，寻找不到直接证据返回 false。
> 21. `is_fractional_share`：拍卖的标的物是否为“部分产权”（例如：某某房屋 50% 的份额、二分之一产权）？如果是部分产权，请务必返回 true，否则返回 false。
> 22. `tax_is_company_owned`：标的物的原所有人（即被执行人）是否为一家“公司”、“企业法人”或者挂在企业名下？如果是，返回 true（意味着买受人需承担极高的土地增值税），如果是个人则返回 false。
> 23. `has_lease_before_mortgage`：公告中是否有明确表述如：“该租赁关系设立于抵押权之后”、“不能对抗抵押权”、“法院负责带租清场”？如果是这种**其实可以强制赶走租客的“假长租”**，必须精准识别并返回 true；如果是普通无法清场的租约，或没有租约，均返回 false。
>
