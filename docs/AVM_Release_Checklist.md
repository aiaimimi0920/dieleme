# AVM 上线门槛与发布检查清单

> 发布标记（首发）：`AVM v1.0-baseline`

本文档定义 AVM 首次上线前必须满足的三类硬性门槛：字段完整率、评估误差、接口稳定性；并提供标准发布/回滚步骤。

---

## 1. 上线硬性门槛（Go/No-Go）

### 1.1 字段完整率（Data Completeness）

**检查对象**：Canonical 数据集中用于估值与风控的关键字段。  
**统计窗口**：最近 7 天待评估样本，且样本量 `N >= 1000`。

#### A. 估值核心字段（必须项）
- `item_id`
- `auction_date`
- `latitude`
- `longitude`
- `transaction_price`
- `area_sqm`
- `housing_type`

**门槛**：
- 单字段完整率均 `>= 99.0%`
- 核心字段联合完整率（同一条记录同时具备以上字段）`>= 97.0%`

#### B. 风控核心字段（必须项）
- `is_occupied`
- `has_long_lease`
- `clear_delivery`
- `tax_burden`
- `is_fractional_share`

**门槛**：
- 单字段完整率均 `>= 95.0%`
- 风控字段联合完整率 `>= 92.0%`

#### C. 阻断规则
任一项低于门槛即 **禁止上线**，进入数据修复与重跑流程。

---

### 1.2 评估误差（Model Error）

**评估口径**：使用时间切分的离线验证集（最近 30 天未参与训练样本）。  
**价格口径**：以 `actual_paid_price` 或 `transaction_price`（按当前可用最高优先级）作为真实值。

**门槛**：
- `MAPE <= 12%`
- `P50 APE <= 8%`
- `P90 APE <= 25%`
- 不允许出现系统性偏差：
  - 按行政区分桶后，任一区域平均偏差绝对值 `> 10%` 视为不通过

> 说明：首发版本以稳健为优先，不追求极限拟合；后续版本可逐步收紧门槛（例如 MAPE <= 10%）。

---

### 1.3 接口稳定性（API Stability）

**检查对象**：对外估值接口（含批量与单条查询）。  
**压测窗口**：连续 24 小时 + 峰值 1 小时压测。

**门槛**：
- 可用性（Availability）`>= 99.9%`
- 错误率（5xx）`<= 0.1%`
- `P95` 延迟 `<= 800ms`
- `P99` 延迟 `<= 1500ms`
- 超时率 `<= 0.2%`
- 发布后 24 小时内无 Sev-1 / Sev-2 事故

#### 契约稳定性要求
- 响应 JSON 字段不删减、不改名
- 新增字段只能“向后兼容追加”
- 错误码语义保持兼容（不可复用旧错误码表达新含义）

## 2. `AVM v1.0-baseline` 发布清单

### 2.1 发布前（T-1）
- [ ] 完成字段完整率报表并归档
- [ ] 完成离线误差评估并归档
- [ ] 完成接口压测与稳定性报告
- [ ] 产出发布变更说明（模型版本、特征版本、数据快照日期）
- [ ] 确认监控看板与报警阈值已生效

### 2.2 发布中（T）
- [ ] 使用灰度策略：5% → 20% → 50% → 100%
- [ ] 每个阶段至少观察 30 分钟
- [ ] 核验实时指标：QPS、P95、5xx、超时率、业务命中率
- [ ] 异常触发即停止扩容并执行回滚

### 2.3 发布后（T+1）
- [ ] 复盘发布窗口内告警
- [ ] 对比灰度前后误差与稳定性漂移
- [ ] 归档 `AVM v1.0-baseline` 发布记录（时间、负责人、证据链接）

## 3. 回滚说明（必须保留）

当触发以下任一条件时，立即回滚到上一稳定版本：
- 核心 SLA 指标连续 10 分钟不达标
- 5xx 错误率超过 0.5% 且持续 5 分钟
- 出现 Sev-1 / Sev-2 事故
- 估值结果出现明显系统性漂移（如大面积高估/低估）

### 标准回滚步骤
1. **流量切换**：网关权重切回上一稳定版本（或关闭新版本路由）。
2. **模型回退**：将在线模型版本从 `AVM v1.0-baseline` 回退至 `last_stable`。
3. **数据回退**：恢复上一版特征快照与配置（如特征映射、阈值表）。
4. **健康检查**：确认 SLA、错误率、延迟恢复正常后解除故障态。
5. **冻结发布**：问题未完成 RCA 前，禁止再次发布。

### 回滚后必做
- 30 分钟内同步事故通告（影响面、回滚时间、当前状态）
- 24 小时内提交 RCA（根因、修复项、预防项、负责人、截止日期）
